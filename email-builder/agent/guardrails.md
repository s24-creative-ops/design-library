# Guardrails

- `preview/` und `email/` gewinnen immer gegen `agent/`.
- Keine neuen Modul- oder Template-Strukturen frei erfinden, wenn bestehende Projektmuster ausreichen.
- Keine Designaenderungen ohne ausdrueckliche User-Anweisung.
- Keine Live-Iterable-Validierung in diesem Schritt behaupten.
- Preview darf nie die finale E-Mail-Wahrheit ersetzen.
- `email/` darf nie von einem alten Spiegelordner oder Alt-Export abgeleitet werden, wenn die aktive Datei schon vorliegt.
- Wenn ein Mapping, eine Shell oder eine Regel noch unklar ist, offen markieren statt still zu glaetten.
- Ausgeschlossene Legacy-Holds nicht nebenbei in den Kernpfad ziehen.
